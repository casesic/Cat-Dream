我一直很喜欢 Markdown 的原因，是它让写作这件事保持克制。

你不需要打开厚重的后台，也不需要在富文本编辑器里和格式较劲。文章就是文件，文件就是内容本身。

## 适合个人网站的原因

Markdown 很适合下面这些场景：

- 技术博客
- 学习笔记
- 项目复盘
- 周报或随笔

如果以后需要更丰富的能力，也可以慢慢把 Markdown 升级成 MDX，让内容里直接嵌 React 组件。

## 配合 SSG 的体验

当我们在构建阶段读取 Markdown 文件时，站点会生成纯静态页面。这意味着：

- 部署更快
- 成本更低
- SEO 更稳定
- 页面结构更清晰
